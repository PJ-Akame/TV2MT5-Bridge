# MarketHoursDisplay

チャート上に **東京・ロンドン・NY** の市場帯（＋任意の第4セッション）と **危険帯・単独注意帯** を **色付きブロック** で表示します。**表示されるチャート時刻軸は常に取引サーバー時刻**です。**入力時刻**は `InpClock` で **サーバー時刻** か **日本時間（JST）** を選べます。注文ロジックは含みません。

## ファイル

| ファイル | 説明 |
|----------|------|
| `MarketHoursDisplay.mq5` | カスタムインジケーター（v2.20） |

## インストール

1. MetaTrader 5 を起動
2. `File` → `Open Data Folder`
3. `MQL5\Indicators\` に `MarketHoursDisplay.mq5` をコピー（またはシンボリックリンク）
4. MetaEditor で開き **Compile**
5. ナビゲーターの **インディケーター** からチャートにドラッグ

## 表示の考え方

### `InpClock = サーバー時刻`（従来）

- **東京・ロンドン・NY**: 開始・終了は **入力＝サーバー時刻**。**危険帯**は ON のとき **その日の市場開始（ロンドン/NY は ST 反映後の開始）から 1 時間** 固定です（時刻入力なし）。
- **ロンドン `London_DST` / NY `NY_DST`**: ON のとき、**描画する各日の暦日** で「夏側」とみなす期間では **市場の開始・終了** だけ +1 時間します。危険帯は **その補正後の開始から 1 時間** です。
  - **ロンドン**: 3 月の最終日曜日以降かつ 10 月の最終日曜日**未満**の日（英国 BST の簡易モデル）。
  - **NY**: 3 月の第 2 日曜日以降かつ 11 月の第 1 日曜日**未満**の日（米東部 DST の簡易モデル）。
- **東京**: 日本のサマータイムは無いため、常に入力どおりです。

### `InpClock = 日本時間(JST)`

- **東京・ロンドン・NY** は **「市場時間(JST)」** グループの時刻（既定は東京 9:00〜10:00、ロンドン夏/冬・NY 夏/冬の2段など）を **サーバー暦上の同等の絶対時刻** に換算してブロックを描画します。換算は `TimeTradeServer() - TimeGMT()` と **JST = UTC+9** から、**同一瞬間で JST の壁時計がサーバーより何時間進んでいるか** を求め、その差でずらします（例: 日本 22:00 とチャート 16:00 が一致するのは、サーバーが JST より約 6 時間遅い場合）。
- ロンドン/NY の **夏冬** は、暦日ごとに英国 BST・米東部 DST の簡易ルールで切り替え、**JST で別々に入力した開始・終了** を使います（`London_DST` / `NY_DST` の +1h は **JST モードでは使いません**）。
- **Session 4 / X1 / X2** は **常にサーバー時刻** の入力のままです（JST 換算の対象外）。

### 共通

- **Session 4**: 任意の追加セッション（例: シドニー）。サーバー入力では ST 補正なし。**危険帯**は入力の開始・終了時刻（東京等とは別）。
- **単独注意帯 X1/X2**: セッション外の矩形（例: ロール付近）。

終了が開始より同日内で小さい場合は **翌日まで** に伸ばします（例: 22:00～翌 07:00）。

## 主な入力

### 共通

- **InpClock**: **サーバー時刻** または **日本時間(JST)**。JST のとき市場ブロックは上記の換算で描画します。
- **InpDaysBack / InpDaysForward**、**InpRefreshSec**、凡例関連

### 下部国名ラベル

- **InpShowCountryFooter**: 国名テキスト（`OBJ_TEXT`）を表示するか。**横位置はセッション開始＝文字列の左端**。
- **InpFooterLiftPct**: チャートに **いま表示されている** 価格レンジのうち、**下端から価格幅の何％上**に国名ラベル **左下端** のアンカーを置くか（既定 2.5）。市場の **開始バー時刻＝ラベル文字の左端**（`ANCHOR_LEFT_LOWER`）に揃えます。
- **Tokyo_CountryLabel** など: 既定は **日本 / 英国 / 米国**。Session 4 は **S4_CountryLabel** が空なら国名なし（任意で例: オーストラリア）。
- **InpFooterFontSize / InpFooterColor**: 国名の見た目。

### 東京市場 / ロンドン市場 / NY市場（`InpClock = サーバー` のとき）

- 有効化、名称、開始・終了（HH:MM・サーバー時刻）、色、透明度、**危険**の ON/OFF・色・透明度（内容は **開始から 1 時間**。ロンドン/NY は `*_DST` 列挙より **補正後の開始** が基準）
- **London_DST**、**NY_DST**: 上記サマータイム補正の ON/OFF

### 市場時間(JST)（`InpClock = JST` のとき）

- 東京・ロンドン（夏/冬）・NY（夏/冬）の **HH:MM**。ロンドン/NY は日ごとの BST・米DST 簡易ルールで夏行または冬行を選びます。終了が翌日にまたがるときは終了 ≤ 開始として入力（例: 23:30〜0:30）。

### Session 4、X1 / X2

- **サーバー時刻** の入力（JST モードでも換算しません）。

## 注意

- ST の境界は **暦日ベースの近似** です。移行日の深夜付近は実際の公式ルールと 1 時間ずれる場合があります。
- **サーバー時刻** 入力モードでは、ベースの HH:MM は冬寄りで合わせ、ST 期間中だけ +1h で意図に近づける想定です。
- **JST** モードでは `TimeTradeServer() - TimeGMT()` に依存します。**週末・メンテ・接続異常時**はサーバーが GMT オフセットを更新していないように見えることがあり、再読み込みや翌営業日に確認してください。
- 「危険」は運用の目安です。
